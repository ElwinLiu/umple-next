# Security Policy

We welcome reports of security vulnerabilities in UmpleOnline — including the web frontend, the backend API, the sandboxed code execution service, and the Umple compiler itself. We will endeavour to fix issues promptly, although as an open source project, the ability to deliver depends on the availability of volunteer contributors.

## Supported Versions

Only the latest deployed version is supported. Since UmpleOnline is continuously deployed from `master`, please verify your report against the live instance at [umpleonline.org](https://umpleonline.org/) before reporting.

## Reporting a Vulnerability

Report a vulnerability by emailing the project manager at **timothy.lethbridge@uottawa.ca**.

Please include:
- A description of the vulnerability and its potential impact
- Steps to reproduce the issue
- Any relevant Umple code, API requests, or screenshots

Please provide a real return email address so we can contact you for more information.

**Do not open a public GitHub issue for security vulnerabilities.**
